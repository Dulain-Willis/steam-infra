# Doc Writing Instructions

## Audience
Write for someone with **zero prior knowledge**. No working knowledge to build from.

## Core Rule: Leave Nothing on the Table
If you introduce a concept — any concept — you must explain it before moving on.

Example: If you say "the prefix length controls how many bits are fixed," you must stop and explain what bits are before continuing. Do not assume the reader can look it up or fill in gaps.

## Checklist Before Publishing Any Explanation
- Did I introduce a term without defining it?
- Did I use an analogy that itself requires background knowledge?
- Would a person who has never touched networking / cloud / Linux understand every sentence?

If no to any of those — rewrite.

## Tone
- Conversational, not textbook
- Short sentences
- Build one idea at a time before stacking the next on top
